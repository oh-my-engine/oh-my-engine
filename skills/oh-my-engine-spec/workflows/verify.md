# Spec Verify Workflow

## Goal

Prove the implementation satisfies the change and does not drift from the capability spec.

## Execution Steps

### Step 1: Check implementation evidence
```
1. Validate completed tasks
2. Run relevant tests and checks
3. Confirm acceptance criteria are satisfied
```

### Step 2: Check spec alignment
```
1. Confirm spec delta matches code behavior
2. Record residual risk
3. Note anything that blocks archive
```
