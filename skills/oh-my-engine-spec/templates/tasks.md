# Implementation Tasks

## Execution Rules
- Keep tasks small, reviewable, and testable.
- Each completed task should leave the repo in a working state.

## Intake Context
- Review `openspec/changes/<change-slug>/context/source.md` before rewriting requirements.
- Review `openspec/changes/<change-slug>/context/prompt.md` to preserve operator intent.
- Convert multimodal findings from `openspec/changes/<change-slug>/context/analysis.md` into executable tasks.

## Phase 1: Discovery and Setup
- [ ] Confirm impacted files and capabilities
- [ ] Update or create relevant specs under `openspec/changes/<change-id>/specs/`

## Phase 2: Implementation
- [ ] Implement the change
- [ ] Add or update tests
- [ ] Update docs or developer guidance if behavior changed

## Phase 3: Verification
- [ ] Run focused verification
- [ ] Check acceptance criteria
- [ ] Record remaining risks
