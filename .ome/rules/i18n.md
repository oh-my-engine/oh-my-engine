# 多语言规则（i18n）

## 强制要求

### 1. 所有用户可见文本必须使用 i18n

```typescript
// ❌ 禁止
<Text>登录</Text>
<Text text="Submit" />

// ✅ 必须
<Text tx="auth:login" />
const { t } = useTranslation()
const msg = t("feedback:saveSuccess")
```

### 2. 必须提供 4 种语言翻译

- **en**: English（英语）
- **zh-CN**: 简体中文
- **zh-TW**: 繁体中文
- **th**: ไทย（泰语）

### 3. 翻译键命名规范

格式：`<module>:<screen>.<element>`

示例：
- `auth:login.title` - 登录页面标题
- `profile:settings.saveButton` - 设置页面保存按钮
- `common:button.confirm` - 通用确认按钮

### 4. React 组件内使用 useTranslation hook

```typescript
// ✅ 正确
const { t } = useTranslation()
const message = t("feedback:saveSuccess")

// ❌ 禁止（在组件内）
import { translate } from "app/i18n"
const message = translate("feedback:saveSuccess")
```

### 5. 语言切换后数据必须重载

- 后端 API 自动携带 `Accept-Language` header
- 切换语言后，所有显示的数据需要重新请求

## 验证清单

生成代码后必须检查：

- [ ] 所有用户可见文本都使用了 i18n
- [ ] 提供了 4 种语言的翻译
- [ ] 翻译键命名符合规范
- [ ] 使用了 `useTranslation()` hook（组件内）
