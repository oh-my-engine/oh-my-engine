# 多语言规则（i18n）

## 强制要求

### 1. 所有用户可见文本必须使用 i18n

```typescript
// ❌ 禁止
<Text>登录</Text>
<Text text="Submit" />

// ✅ 必须
<Text tx="auth:login" />
const { t } = useTranslation()
const msg = t("feedback:saveSuccess")
```

### 2. 必须提供 4 种语言翻译

- **en**: English（英语）
- **zh-CN**: 简体中文
- **zh-TW**: 繁体中文
- **th**: ไทย（泰语）

### 3. 翻译键命名规范

格式：`<module>:<screen>.<element>`

示例：
- `auth:login.title` - 登录页面标题
- `profile:settings.saveButton` - 设置页面保存按钮
- `common:button.confirm` - 通用确认按钮

### 4. React 组件内使用 useTranslation hook

```typescript
// ✅ 正确
const { t } = useTranslation()
const message = t("feedback:saveSuccess")

// ❌ 禁止（在组件内）
import { translate } from "app/i18n"
const message = translate("feedback:saveSuccess")
```

### 5. 语言切换后数据必须重载

- 后端 API 自动携带 `Accept-Language` header
- 切换语言后，所有显示的数据需要重新请求

## 翻译质量标准

### 1. 使用上下文相关的翻译

```json
// ❌ 直译
{
  "en": "Save",
  "zh-CN": "保存",
  "zh-TW": "保存",
  "th": "บันทึก"
}

// ✅ 上下文相关
{
  "en": "Save Changes",
  "zh-CN": "保存更改",
  "zh-TW": "儲存變更",
  "th": "บันทึกการเปลี่ยนแปลง"
}
```

### 2. 保持语气一致

- 使用友好、专业的语气
- 避免过于正式或过于随意
- 保持品牌调性

### 3. 注意文化差异

- 日期格式：en (MM/DD/YYYY), zh (YYYY-MM-DD), th (DD/MM/YYYY)
- 货币符号：根据语言显示对应货币
- 数字格式：千位分隔符等

## 验证清单

生成代码后必须检查：

- [ ] 所有用户可见文本都使用了 i18n
- [ ] 提供了 4 种语言的翻译
- [ ] 翻译键命名符合规范
- [ ] 使用了 `useTranslation()` hook（组件内）
- [ ] 翻译质量符合标准（非直译）
- [ ] 考虑了文化差异

## 相关文件

- 翻译文件：`app/i18n/[lang]/`
- i18n 配置：`app/i18n/i18n.ts`
- 翻译工具：`scripts/i18n/`
